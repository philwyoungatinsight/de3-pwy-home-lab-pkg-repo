`known-fw-repos.yaml` and visualises repos as compound nodes containing

## Data Source

`config/tmp/fw-repos-visualizer/known-fw-repos.yaml` — pre-generated by
`fw-repos-visualizer --list`. The GUI reads it at load time and on "Refresh".
It never writes to it. If the file is absent, the view shows an empty graph
with a hint to run fw-repos-visualizer.

YAML schema (current):
```yaml
data:
  repos:
    <repo-name>:
      url: <str|empty>
      created_by: <str|empty>   # lineage: which repo declared this one
      source: local|cloned|declared
      settings_dirs:
        - path: <str>
          packages:
            - name: <str>
              package_type: external|embedded
              exportable: true|false
```

## Cytoscape Element Schema

### Repo compound nodes

```python
{
    "data": {
        "id":          "<repo-name>",          # e.g. "de3-runner"
        "label":       "<repo-name>",
        "node_type":   "repo",
        "source":      "local|cloned|declared",
        "url":         "<url or empty>",
        "created_by":  "<parent-repo or empty>",
    },
    # no "parent" key → top-level compound node
    # "position": {"x": ..., "y": ...}  ← injected when preset layout active
}
```

### Package child nodes

Package appears inside every repo that lists it, with `package_type` as a
qualifier — no deduplication across repos.

```python
{
    "data": {
        "id":           "<repo-name>/<pkg-name>",  # e.g. "de3-runner/aws-pkg"
        "label":        "<pkg-name>",
        "node_type":    "package",
        "pkg_type":     "embedded|external",       # qualifier shown in label/badge
        "exportable":   True|False,
        "parent":       "<repo-name>",             # Cytoscape compound parent
    },
    # "position": {"x": ..., "y": ...}  ← injected when preset layout active
}
```

Label strategy when metadata is visible:
```
aws-pkg
[embedded] [exportable]
```
Implemented via `label` field updated in the computed_var based on
`fw_repos_show_pkg_meta` state flag (see § State fields).

### Lineage edges

Added for every repo where `created_by` is non-empty. Only emitted when
`fw_repos_show_lineage` is True.

```python
{
    "data": {
        "id":        "<parent>→<child>",
        "source":    "<created_by value>",   # parent repo
        "target":    "<repo-name>",          # child repo
        "edge_type": "lineage",
    },
}
```

## State Fields

```python
# Data
framework_repos_data:         dict = {}    # raw repos dict from known-fw-repos.yaml
fw_repos_positions:           dict = {}    # {node_id: {x, y}} from fw-repos-layout.yaml
fw_repos_root_ids:            list = []    # repo names with no created_by (lineage roots)

# Collapse state (persisted to current.yaml)
fw_repos_collapsed_repos:     list = []    # repo names with packages hidden

# Layout selector (persisted to current.yaml)
fw_repos_layout:              str  = "cose"   # "cose"|"breadthfirst"|"dagre"|"preset"

# Appearance options — all persisted to current.yaml under flat fw_repos_* keys
fw_repos_show_lineage:        bool = True   # lineage edges between repos
fw_repos_show_source_badge:   bool = True   # source badge on repo nodes
fw_repos_show_url:            bool = False  # URL in repo node label
fw_repos_show_packages:       bool = True   # master switch: show package child nodes
fw_repos_show_pkg_type_badge: bool = True   # embedded/external badge in pkg label
fw_repos_show_exportable:     bool = True   # exportable badge in pkg label
fw_repos_merge_duplicates:    bool = False  # v1: no-op; future dedup option
```

`fw_repos_collapsed_repos` excludes package children for the listed repos.
`fw_repos_show_packages = False` is the master switch — equivalent to
collapsing all, but kept separate so it doesn't overwrite the per-repo list.

## Layout Algorithms

Four choices exposed in the menu bar:

| Name | Cytoscape name | Notes |
|------|---------------|-------|
| Force (default) | `cose` | Good organic layout; no extra library |
| Tree | `breadthfirst` | Hierarchical top-down using lineage edges; built into Cytoscape |
| Layered | `dagre` | Strict left-right DAG layers; requires `cytoscape-dagre` npm package |
| Saved | `preset` | Uses positions from `fw-repos-layout.yaml`; hidden unless layout file exists |

For `breadthfirst` and `dagre`, set the layout on **repo-level nodes only**
(filtered graph with compound nodes and lineage edges, no package children).
After layout, child positions are auto-managed by Cytoscape's compound sizing.

`dagre` requires adding `"cytoscape-dagre@2.5.0"` to `lib_dependencies` on
`_CytoscapeGraph` (or a new subclass `_FwReposCytoscapeGraph` to avoid
touching the existing component). The `dagre` npm package is a peer dep.
Prefer a subclass to avoid touching the existing infra Cytoscape component.

Layout is applied via the `layout` prop. When the user switches layout from
the dropdown, set `fw_repos_layout` state; the computed `fw_repos_cyto_layout`
var returns the correct dict:

```python
@computed_var
def fw_repos_cyto_layout(self) -> dict:
    m = self.fw_repos_layout
    if m == "preset" and self.fw_repos_positions:
        return {"name": "preset", "fit": True, "padding": 80}
    if m == "breadthfirst":
        return {"name": "breadthfirst", "directed": True, "padding": 80,
                "spacingFactor": 1.4, "roots": "#<local-repo>"}
    if m == "dagre":
        return {"name": "dagre", "rankDir": "LR", "padding": 80,
                "nodeSep": 50, "rankSep": 120}
    # default: cose
    return {"name": "cose", "animate": False, "padding": 80,
            "nodeRepulsion": 8000, "idealEdgeLength": 150}
```

For `breadthfirst`, `roots` should be the repo(s) with no `created_by` (i.e.
the roots of the lineage tree). Identify at element-build time and store as
`fw_repos_root_ids: list[str]` in AppState, then splice into the layout dict.

### Appearance dropdown

All items are checkboxes rendered as `rx.dropdown_menu.checkbox_item` (or
equivalent Radix component). Grouped into labelled sections:

```
Repos
  ☑  Show lineage edges
  ☑  Show source badge  (local / cloned / declared)
  ☐  Show repo URL

Packages
  ☑  Show packages
  ☑  Show package type badge  (embedded / external)
  ☑  Show exportable badge
  ☐  Show duplicates as merged node  (future: single node shared across repos)

(future: Capabilities section once YAML includes capability data)
```

Every item maps 1:1 to an AppState bool. The "Show packages" toggle is the
master switch — when off, all package child nodes are excluded from
`fw_repos_cytoscape_elements` regardless of other toggles (equivalent to
collapsing all repos at once, but kept as a separate display flag so it
doesn't clear the per-repo collapse list).

"Show duplicates as merged node" is a **future** option (greyed out in v1,
no-op). When enabled it would deduplicate package nodes that appear in
multiple repos into a single shared node with edges from each repo compound.
Exposed as an Appearance option so the user can switch between both views
rather than it being a fixed design decision.

## computed_var: `fw_repos_cytoscape_elements`

```python
@computed_var
def fw_repos_cytoscape_elements(self) -> list[dict]:
    repos        = self.framework_repos_data
    positions    = self.fw_repos_positions if self.fw_repos_layout == "preset" else {}
    collapsed    = set(self.fw_repos_collapsed_repos)
    show_pkgs    = self.fw_repos_show_packages       # master switch
    show_lin     = self.fw_repos_show_lineage
    show_src     = self.fw_repos_show_source_badge
    show_url     = self.fw_repos_show_url
    show_type    = self.fw_repos_show_pkg_type_badge
    show_export  = self.fw_repos_show_exportable

    nodes: list[dict] = []
    edges: list[dict] = []

    for repo_name, repo in repos.items():
        url    = repo.get("url", "") or ""
        source = repo.get("source", "")
        label_parts = [repo_name]
        if show_src and source:
            label_parts.append(f"[{source}]")
        if show_url and url:
            label_parts.append(url)
        node: dict = {"data": {
            "id":         repo_name,
            "label":      "\n".join(label_parts),
            "node_type":  "repo",
            "source":     source,
            "url":        url,
            "created_by": repo.get("created_by", "") or "",
        }}
        if repo_name in positions:
            node["position"] = positions[repo_name]
        nodes.append(node)

        # Package children: skip if master switch off OR repo is collapsed
        if show_pkgs and repo_name not in collapsed:
            for sdir in repo.get("settings_dirs", []):
                for pkg in sdir.get("packages", []):
                    child_id   = f"{repo_name}/{pkg['name']}"
                    pkg_type   = pkg.get("package_type", "external")
                    exportable = pkg.get("exportable", False)
                    badges = []
                    if show_type:
                        badges.append(f"[{pkg_type}]")
                    if show_export and exportable:
                        badges.append("[exportable]")
                    label = pkg["name"] + ("\n" + " ".join(badges) if badges else "")
                    child: dict = {"data": {
                        "id":         child_id,
                        "label":      label,
                        "node_type":  "package",
                        "pkg_type":   pkg_type,
                        "exportable": 1 if exportable else 0,  # Cytoscape selector compat
                        "parent":     repo_name,
                    }}
                    if child_id in positions:
                        child["position"] = positions[child_id]
                    nodes.append(child)

        if show_lin and repo.get("created_by"):
            edges.append({"data": {
                "id":        f"{repo['created_by']}→{repo_name}",
                "source":    repo["created_by"],
                "target":    repo_name,
                "edge_type": "lineage",
            }})

    return nodes + edges
```

## Event Handlers

```python
def save_fw_repos_layout(self, positions: dict):
    self.fw_repos_positions = positions
    self.fw_repos_layout = "preset"
    _FW_REPOS_LAYOUT.parent.mkdir(parents=True, exist_ok=True)
    _FW_REPOS_LAYOUT.write_text(
        yaml.dump({"positions": positions}, default_flow_style=None, sort_keys=True)
    )

def reset_fw_repos_layout(self):
    self.fw_repos_positions = {}
    self.fw_repos_layout = "cose"
    if _FW_REPOS_LAYOUT.exists():
        _FW_REPOS_LAYOUT.unlink()

def toggle_fw_repo_collapsed(self, repo_name: str):
    c = list(self.fw_repos_collapsed_repos)
    if repo_name in c:
        c.remove(repo_name)
    else:
        c.append(repo_name)
    self.fw_repos_collapsed_repos = c

def collapse_all_fw_repos(self):
    self.fw_repos_collapsed_repos = list(self.framework_repos_data.keys())

def expand_all_fw_repos(self):
    self.fw_repos_collapsed_repos = []

async def refresh_fw_repos_data(self):
    import subprocess
    fw_viz = str(_GIT_ROOT / "infra/_framework-pkg/_framework/_fw-repos-visualizer/fw-repos-visualizer")
    subprocess.run([fw_viz, "--list"], capture_output=True)
    if _FW_REPOS_YAML.exists():
        raw = yaml.safe_load(_FW_REPOS_YAML.read_text()) or {}
        self.framework_repos_data = raw.get("data", {}).get("repos", {})
```

# Plan: Revamp Git Repos — Fix Symlinks, Move Config, Retire Legacy Repos

## Objective

Complete the git repo refactoring: fix all broken symlinks in the generated framework repos
(`~/git/de3-generated-framework-repos/`), move the `framework_repo_manager.yaml` to the

## Context

### Root cause of broken symlinks

Every generated repo (e.g. `de3-pwy-home-lab-pkg-repo`) has this symlink chain:

```
README.md  →  infra/_framework-pkg/_framework/_git_root/README.md
                      ↓
infra/_framework-pkg  →  ../_ext_packages/de3-runner/main/infra/_framework-pkg
                                  ↓
_ext_packages/de3-runner/main  →  /home/pyoung/git/de3-ext-packages/de3-runner/main
                                             ↓
                                   DOES NOT EXIST ← root cause
```

`de3-runner` lives at `~/git/de3-runner/main` but `framework_package_management.yaml` sets
`external_package_dir: 'git/de3-ext-packages'`, so `pkg-mgr` always creates `_ext_packages`
symlinks pointing into `~/git/de3-ext-packages/<slug>/main`. All other repos ARE in
`de3-ext-packages`; only `de3-runner` is missing because it was never cloned there —
it pre-dates the ext-packages convention.

Confirmed:
- `~/git/de3-ext-packages/` exists and contains all 11 generated repos as real `main/` directories.
- `~/git/de3-ext-packages/de3-runner/` does NOT exist.
- `~/git/de3-runner/main/` exists and is the working de3-runner clone.

### Secondary broken symlink: `.gitlab-ci.yml`

Every generated repo has:
```
.gitlab-ci.yml  →  infra/default-pkg/_framework/_git_root/.gitlab-ci.yml
```
`default-pkg` is pruned by `fw-repo-mgr` so this symlink is always dangling. This is a
`fw-repo-mgr` bug — addressed in a separate step below.

### The `de3-framework-pkg-repo` is already working

`de3-framework-pkg-repo` has `_framework-pkg` as an **embedded** package (real directory),
so its top-level symlinks resolve correctly without any de3-runner dependency. Only the
`.gitlab-ci.yml` symlink is broken there.

### `de3-ext-packages/<repo>/main/` raw clones also have broken symlinks

The raw clones pulled from GitHub (e.g. `de3-ext-packages/de3-pwy-home-lab-pkg-repo/main/`)
have broken `infra/_framework-pkg` symlinks because those clones have no `_ext_packages/`
sub-directory. This does NOT block normal use: consumer repos only need the specific package
directory (e.g. `infra/pwy-home-lab-pkg/`) to be accessible. The top-level file symlinks
in these raw clones are cosmetic and do not affect package import.

### Framework_repo_manager.yaml is currently in pwy-home-lab-pkg only

The config that defines all 13 generated repos lives at:
```
~/git/pwy-home-lab-pkg/infra/pwy-home-lab-pkg/_config/_framework_settings/framework_repo_manager.yaml
```
It should live in `de3-central-index-repo` so that the central index is the canonical
place to generate or rebuild any repo in the ecosystem.

### All generated repos still reference `de3-runner` as source of `_framework-pkg`

Every generated repo's `framework_packages.yaml` has:
```yaml
- name: _framework-pkg
  package_type: external
  repo: de3-runner
  source: https://github.com/philwyoungatinsight/de3-runner.git
  git_ref: main
```
Retiring `de3-runner` requires changing this to point at `de3-framework-pkg-repo`. This is
a multi-repo change and is scoped as a future step, NOT done in this plan.

### Step 2 — Fix the `.gitlab-ci.yml` broken symlink in all generated repos

Each generated repo has `.gitlab-ci.yml → infra/default-pkg/_framework/_git_root/.gitlab-ci.yml`
which is always dangling because `fw-repo-mgr` prunes `default-pkg`.

**Fix option A** (recommended): Remove the broken symlink from every generated repo and commit.
```bash
for repo in ~/git/de3-generated-framework-repos/*/; do
  link="$repo/.gitlab-ci.yml"
  if [ -L "$link" ] && [ ! -e "$link" ]; then
    rm "$link"
    git -C "$repo" add -A
    git -C "$repo" commit -m "chore: remove broken .gitlab-ci.yml symlink" || true
  fi
done
```

**Fix option B** (fix fw-repo-mgr): Update `fw-repo-mgr`'s rsync/template step to not create
a `.gitlab-ci.yml` symlink when `default-pkg` is not present in the target repo. This prevents
the issue for future builds but doesn't remove existing broken symlinks.

Recommend doing both: remove now (A), fix fw-repo-mgr (B) so it doesn't recur.

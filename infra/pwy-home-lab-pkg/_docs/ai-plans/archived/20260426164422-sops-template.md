# Plan: Populate framework_settings_sops_template and Wire fw-repo-mgr

## Objective

Repurpose the commented-out `framework_settings_sops_template` key in
`framework_repo_manager.yaml` to hold the content for `.sops.yaml` at the git root
of each generated repo. Update `fw-repo-mgr` to write this content as `.sops.yaml`
after building each repo. Use the existing `pwy-home-lab-pkg` `.sops.yaml` as the
default.

## Context

- `framework_settings_sops_template` exists in the YAML but is commented out. Its
  current placeholder (GCP billing/sa secrets) is wrong for this purpose — the
  `.sops.yaml` file is not secret (PGP fingerprints are public); it belongs in the
  plaintext config.
- `fw-repo-mgr` has no code that reads `framework_settings_sops_template`. The
  existing comment at line 391 ("The same pattern applies to SOPS overrides…")
  describes an intended but unimplemented feature.
- Currently `.sops.yaml` in generated repos is inherited via `rsync -a` from the
  de3-runner source template (line 479). The de3-runner template's `.sops.yaml` is a
  symlink → `infra/_framework-pkg/_framework/_git_root/.sops.yaml`. Its content is
  identical to the pwy-home-lab-pkg root `.sops.yaml`, so using the latter as the
  default is correct.
- `fw-repo-mgr` lives at
  `_ext_packages/de3-runner/main/infra/_framework-pkg/_framework/_fw-repo-mgr/fw-repo-mgr`
  (the symlink resolves here). Changes to it are in the de3-runner cached clone — this
  is acceptable since de3-runner is the framework source.

```yaml
  # Template written as .sops.yaml at the git root of every generated repo.
  # Content is not secret (PGP fingerprints are public) — lives in plaintext config.
  # fw-repo-mgr writes this after rsync/pull, overriding any .sops.yaml inherited
  # from the source template.
  framework_settings_sops_template:
    stores:
      yaml:
        indent: 2
    creation_rules:
      ################################################################################
      # Package-specific overrides — must come BEFORE the general rule below.
      # SOPS uses the first matching rule; more-specific path_regex goes first.
      ################################################################################

      # Example: use a different set of keys to encrypt sops files for a particular package.
      - path_regex: '.*infra/unifi-pkg/_config/.*\.sops\.yaml$'
        pgp: >-
          446B4127CF4E00C389579F20986E9FAE2FB2BEBD,
          1FAFFDF2C76C758F736178E2B776DF4CEB6B692B

      ################################################################################
      # Default encryption keys for all packages
      # (infra/<pkg>/_config/<pkg>_secrets.sops.yaml)
      ################################################################################
      - path_regex: '.*infra/[^/]+/_config/.*\.sops\.yaml$'
        pgp: >-
          446B4127CF4E00C389579F20986E9FAE2FB2BEBD,
          1FAFFDF2C76C758F736178E2B776DF4CEB6B692B

    # phil: 1FAFFDF2C76C758F736178E2B776DF4CEB6B692B, 446B4127CF4E00C389579F20986E9FAE2FB2BEBD
```

Note: the trailing comment `# phil: ...` will NOT be preserved by `yaml.dump()` when
fw-repo-mgr writes `.sops.yaml` — comments are stripped by the YAML parser. This is
acceptable; the comment is informational only and lives here as the canonical reference.

### `_ext_packages/de3-runner/main/infra/_framework-pkg/_framework/_fw-repo-mgr/fw-repo-mgr` — modify

**Change 1**: Add `_write_sops_yaml()` function after `_write_settings_template()`
(after line 410):

```bash
# ---------------------------------------------------------------------------
# Write .sops.yaml to the repo root from framework_settings_sops_template.
# Runs after rsync/pull so it overrides any .sops.yaml inherited from source.
# No-op if framework_settings_sops_template is absent from config.
# ---------------------------------------------------------------------------
_write_sops_yaml() {   # _write_sops_yaml <repo_dir>
  local repo_dir="$1"
  [[ -z "$FW_MGR_CFG" ]] && return 0

  python3 - "$repo_dir" "$FW_MGR_CFG" <<'PYEOF'
import sys, yaml, pathlib
repo_dir, cfg_path = sys.argv[1], sys.argv[2]
d = yaml.safe_load(pathlib.Path(cfg_path).read_text()) or {}
template = d.get('framework_repo_manager', {}).get('framework_settings_sops_template')
if not template:
    sys.exit(0)
out_path = pathlib.Path(repo_dir) / '.sops.yaml'
out_path.write_text(yaml.dump(template, default_flow_style=False, sort_keys=False))
print(f'  wrote: .sops.yaml')
PYEOF
}
```

**Change 2**: Call `_write_sops_yaml` in `_build_repo()`, after
`_write_framework_packages_yaml` and before the pkg-mgr sync (Step 4). Insert after
the line `echo "Wrote: $fw_pkgs_out"` (after line 535):

```bash
  # Write .sops.yaml from framework_settings_sops_template
  _write_sops_yaml "$repo_dir"
```

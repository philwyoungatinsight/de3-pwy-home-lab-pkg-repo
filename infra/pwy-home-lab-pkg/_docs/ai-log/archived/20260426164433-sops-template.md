# Populate framework_settings_sops_template and Wire fw-repo-mgr

## Summary

Activated `framework_settings_sops_template` in `framework_repo_manager.yaml` with the
SOPS creation rules from the pwy-home-lab-pkg root `.sops.yaml`. Added `_write_sops_yaml()`
to `fw-repo-mgr` so that when building a generated repo the template is written as
`.sops.yaml` at the repo root, overriding whatever was inherited from the de3-runner
source template via rsync.

## Changes

- **`infra/pwy-home-lab-pkg/_config/_framework_settings/framework_repo_manager.yaml`**
  — replaced the wrong commented-out GCP secrets placeholder with an active
  `framework_settings_sops_template` block containing `stores.yaml.indent: 2` and two
  creation rules (unifi-pkg specific + general `infra/[^/]+/_config/` rule), both using
  the same two PGP fingerprints as the root `.sops.yaml`

- **`_ext_packages/de3-runner/main/infra/_framework-pkg/_framework/_fw-repo-mgr/fw-repo-mgr`**
  — added `_write_sops_yaml()` function (reads `framework_settings_sops_template` from
  config, serialises with `yaml.dump`, writes to `<repo_root>/.sops.yaml`); called in
  `_build_repo()` after `_write_framework_packages_yaml` and before pkg-mgr sync;
  updated stale comment on `_write_settings_template` that described the unimplemented
  SOPS feature

## Notes

- Comments in `framework_settings_sops_template` (the `# phil:` key fingerprint line and
  the section headers) are NOT preserved when `yaml.dump()` writes `.sops.yaml` — this
  is expected and acceptable; the canonical reference for the fingerprints lives in the
  config file.
- The `pgp` value is a folded scalar (`>-`) in the YAML config; after round-tripping
  through `yaml.safe_load` + `yaml.dump`, it becomes a plain scalar on one line — SOPS
  accepts both forms.

# Fix fw-repo-mgr — CLI args, TypeError bug, _EPHEMERAL, and Make support

## Summary

Fixed three bugs in `fw-repo-mgr` and added Makefile support per the framework convention.

## Changes

### Bug 1: TypeError: 'NoneType' object is not iterable
- `infra/_framework-pkg/_config/_framework_settings/framework_repo_manager.yaml`: changed
  `framework_repos:` (YAML null) to `framework_repos: []`. YAML parses a bare key with
  no value as null; when this tier-3 config was used, `_list_repos` emitted `null` JSON
  and the build-dispatch one-liner `[... for r in json.load(stdin)]` crashed.
- Added `or []` guards throughout the script wherever `framework_repos` is accessed to
  defend against null values.

### Bug 2: `$_EPHEMERAL` undefined
- Updated `$_EPHEMERAL` → `$_RAMDISK_DIR` in the validate-config flag block per user
  instruction (`_EPHEMERAL_DIR` is now `_RAMDISK_DIR`). The block is harmless since
  `bash` cannot execute a directory and `|| true` swallows the error.

### Bug 3: Positional subcommands
- Replaced bare positional subcommands (`build`, `validate`, `status`, `help`) with
  GNU-style flags: `-b|--build`, `-v|--validate`, `-s|--status`, `-h|--help`, `-f|--force-push`.
- Added "print usage to stderr and exit 1" when no args are provided.

### Rename + Makefile
- Renamed `fw-repo-mgr` → `run` (CLAUDE.md: use `run` when sibling Makefile exists).
- Created `infra/_framework-pkg/_framework/_fw-repo-mgr/Makefile` with targets:
  `build` (`./run --build`), `validate` (`./run --validate`), `status` (`./run --status`).
- Updated `set_env.sh`:
  - `_FW_REPO_MGR` now points to `.../run` (was `.../fw-repo-mgr`).
  - Removed `_fw-repo-mgr` from the `_set_env_update_path` PATH list (entry-point `run`
    is too generic for bare PATH invocation; same reason `_generate-inventory` is excluded).
